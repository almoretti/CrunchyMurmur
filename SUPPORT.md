# Support

## Questions and bug reports

Search [existing issues](https://github.com/almoretti/CrunchyMurmur/issues) and the [troubleshooting guide](docs/troubleshooting.md) first. If the problem remains, open the appropriate GitHub issue template.

Include the CrunchyMurmur version and installation type; operating system, version, architecture, and Linux desktop/session where relevant; transcription engine and model name without credentials; clear reproduction steps; and masked diagnostics when useful.

Do not post API keys, private calendar URLs, recordings, transcripts, notes, or other personal information. This community project does not offer a guaranteed response time.

## Security issues

Do not open a public issue. Follow [SECURITY.md](SECURITY.md) and use GitHub private vulnerability reporting.

## Feature requests

Open a feature request describing the user problem, affected platforms, privacy implications, and the smallest useful outcome. Check the [roadmap](ROADMAP.md) for existing direction.
